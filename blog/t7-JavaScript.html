The Ruby Hash

In Ruby, hashes are types of objects (of the class Hash) that contain data in pairs. Each pair is made up of a key and a value. Keys are unique, and both keys and values can be referenced. You can be flexible with the type of data you can store, such as Strings, Integers, Arrays, and even other Hashes.

example_hash = { :name => "Joe", :age => 27, :location => "Virginia" }

example_hash = { name:"Joe", age: 27, location: "Virginia" }

example_hash = Hash.new {
  name => "Joe",
  age => 27,
  location => "Virginia
}

All three are valid ways to create a hash object. We can access these keys by doing the following:

example_hash = { :name => "Joe", :age => 27, :location => "Virginia" }

example_hash[:name]

=> "Joe"

example_hash[:age]

=> 27

example_hash[:location]

=> "Virginia"

Because Hash objects in Ruby come with their own set of Hash methods already built in, you can do things like find the first matching key if given a certain value, or find out how many key/value pairs exist:

example_hash.key("Joe")

=> :name

example_hash.size

=> 3

JavaScript Objects

In JavaScript, any object already comes with the ability to store data about itself in what looks similar to a Hash, without it being specifically a Hash object. You can initialize any object with its properties, access them, or even add or change them along the life of this object.

var student = { name: "Joe", age: 27, location: "Virginia" };

console.log(student.name);

Joe

Let's add properties to our student object:

student.lastname = "Won";
student.gpa = 3.7;
student.classOf = 2009;

console.log(student)

{ name: 'Joe', age: 27, location: 'Virginia', lastname: 'Won', gpa: 3.7, classOf: 2009 }

So far, it appears that Ruby hashes and JavaScript objects function the same. There is a distinction, however, based on how JavaScript and Ruby both think of objects slightly differently. Here, we didn't have to specify that our JavaScript object was to be some kind of hash. It just knows to give it properties and handle them in certain ways. Whereas, Ruby requires you to create an object of a specific class, in this case a Hash, so that it knows how it should behave. Here we can see how both languages classify our objects:

console.log(typeof student)

object

And in Ruby:

example_hash.class

=> Hash
